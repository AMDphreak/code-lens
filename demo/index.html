Where this idea comes from

Technical authors pick one naming convention per code example and hope you recognize the pattern. Readers routinely encounter four referential strategies — didactic, schematic, contextual, and role-labeled — without vocabulary to name them. A lens block keeps syntax fixed while identifiers, strings, and comments rotate to show each strategy.

The four lenses

Lens Also called Foregrounds Trade-off
Didactic Concept-labeled Names that are the lesson Fast; unlike production code
Schematic Abstract placeholders Syntax and structure Clear grammar; weak domain transfer
Contextual Production-shaped Real ecosystem idioms Transferable; must excavate relevance
Role-labeled Structural roles Function inside the example Scaffold between didactic and real

Live demo

Desktop: hover lens tabs (glass indicator slides). Touch: swipe the code panel. Each lens uses a distinct panel background from the active color scheme.

Portable specification

Implementations load three JSON5 files from spec/. This keeps UI behavior, themes, and example content independent of any framework.

  • spec/lens-block.schema.json5 — document schema
  • spec/examples/*.json5 — lens-aligned token data
  • spec/themes.json5 — color schemes (tropical, earth, earthenware, patina, fruits, desert)
  • spec/ui.json5 — interaction + animation constants
